THE PROBLEM
----------------------------------------------------------------
AI agents are proliferating across Web3 — but there is no
standard way to verify who an agent is, whether it can be
trusted, or what its track record is.

Today, AI agents operate without:
  - A permanent, verifiable on-chain identity
  - A trustless reputation system built from real interactions
  - Third-party validation of their capabilities
  - A standard lifecycle (active, paused, terminated)

This creates a trust gap that blocks AI agents from being used
in high-stakes, permissionless, on-chain environments.


----------------------------------------------------------------
THE SOLUTION: CODEX ID
----------------------------------------------------------------
Codex ID is a decentralized registry that gives every AI agent
a verifiable on-chain identity, trust score, and lifecycle —
governed by open standards.

Each AI agent is represented as an ERC-721 NFT (Non-Fungible
Token). The tokenId IS the agentId. Whoever holds the NFT owns
the agent and controls its identity, reputation, and lifecycle.

Three registries work together under the ERC-8004 standard:

  1. IDENTITY REGISTRY (CodexIdentity)
     - Mints the agent as an NFT via register()
     - Stores the agent's URI pointing to registration metadata
     - Issues notarized identity attestations (tamper-proof,
       expirable, revocable) via createAttestation()
     - Implements full BAP-578 NFA lifecycle: pause, unpause,
       terminate, fundAgent, setLogicAddress, executeAction
     - Supports structured BAP-578 metadata: persona,
       experience, voiceHash, animationURI, vaultURI, vaultHash
     - Agent wallet is verifiable via EIP-712 signed proof +
       ERC-1271 smart contract wallet fallback

  2. REPUTATION REGISTRY (ReputationRegistry)
     - Clients (NOT the agent owner) submit scored feedback
       via giveFeedback(agentId, score, tag1, tag2, ...)
     - Feedback is permanently stored on-chain (1-indexed)
     - Agent owner can respond via appendResponse()
     - readAllFeedback() supports filtering by tag and
       revocation status
     - getSummary() returns aggregate score across clients

  3. VALIDATION REGISTRY (ValidationRegistry)
     - Agent owner requests third-party assessment via
       validationRequest(validator, agentId, requestURI, hash)
     - Designated validator responds with a 0-100 score via
       validationResponse()
     - getSummary() returns average score across validators
     - All request/response data is permanently on-chain

STANDARDS COMPLIANCE
----------------------------------------------------------------
  ERC-721        Full implementation via ERC721URIStorage
                 Each AI agent = unique NFT, tokenId = agentId
                 Agent wallet auto-cleared on NFT transfer

  ERC-8004       28/28 requirements — Three-registry pattern
  (Trustless     Identity Registry: register, setAgentURI,
  Agents)        setMetadata, getMetadata, setAgentWallet,
                 getAgentWallet, createAttestation,
                 verifyAttestation, revokeAttestation
                 Reputation Registry: giveFeedback,
                 revokeFeedback, appendResponse,
                 readFeedback, readAllFeedback, getSummary,
                 getLastIndex, getResponseCount, getClients
                 Validation Registry: validationRequest,
                 validationResponse, getValidationStatus,
                 getSummary, getAgentValidations,
                 getValidatorRequests

  BAP-578        17/17 core requirements — Non-Fungible Agent
  (NFA Token     Status enum: Active / Paused / Terminated
  Standard)      State struct: balance, status, owner,
                 logicAddress, lastActionTimestamp
                 AgentMetadata: persona, experience,
                 voiceHash, animationURI, vaultURI, vaultHash
                 Lifecycle: pause(), unpause(), terminate()
                 Funding: fundAgent() payable
                 Logic: setLogicAddress(), executeAction()
                 Events: StatusChanged, LogicUpgraded,
                 AgentFunded, MetadataUpdated, ActionExecuted

  EIP-712        Typed structured data for setAgentWallet —
                 requires the new wallet to sign a typed
                 message proving control

  ERC-1271       Smart contract wallet signature fallback
                 in setAgentWallet()

----------------------------------------------------------------
ARCHITECTURE SUMMARY
----------------------------------------------------------------

  User/App
     |
     v
  CodexIdentity (ERC-721 NFT Registry)
  - register() → mint NFT, agentId = tokenId
  - createAttestation() → on-chain identity proof
  - updateAgentMetadata() → BAP-578 structured data
  - pause() / unpause() / terminate() → lifecycle
  - fundAgent() → BNB deposit for agent operations
     |
     +---> ReputationRegistry
     |     - giveFeedback() → client scores (NOT owner)
     |     - appendResponse() → agent replies
     |     - getSummary() → aggregate reputation score
     |
     +---> ValidationRegistry
           - validationRequest() → owner requests review
           - validationResponse() → validator scores 0-100
           - getSummary() → average validation score

All three registries are linked via ERC-721 ownerOf() —
no separate registration needed in Reputation or Validation.
The NFT IS the agent's identity anchor.
